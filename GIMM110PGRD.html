GIMM 110 Personal Game "Range Day"

For this project I wanted to keep things simple due to my lack of experience, and as for the game play the basic idea was to make my game basically a arcade game. The main game I took inspiration from was space invaders, the big difference being the fact that there is no lose condition other than a timer that is slowly ticking down until it triggers the end screen. The game’s graphics were made in adobe photoshop, the terrain pallet was based loosely on the terrain of the shooting range I go to and the stationary targets were based off of the targets I use there. The idea for the bottles were that friends were throwing them from the sides and perhaps some of them have magic?

My game has 3 (technically 4) targets, the stationary target, a thrown beer bottle and a sporadic beer bottle along with a bottle of champagne that’s shot at the player and steals the player’s ammo if it hits them. In the target’s script everything is set up from the hit logic to the “snaky boy movement” (yes that is what I called it). My friend TJ helped me with setting up a lot of this stuff and he is the one who helped me with getting the snaky boy movement set up. The way it works is you have a max and minimum sin value and the script will randomly generate a number when the chosen target is spawned and then when it starts to move up or down it will move sporadically.

A very big feature I added that the shooter stub didn’t originally have was to make it so the player has limited ammo. I did this with the use of if statements such as an if statement that basically says “if ammo is above zero the player can shoot” then when the player shoots it subtracts one round from what they can shoot. Another thing I added that uses an if statement was a bool called “canShoot” which is used to make it so that when time runs out the player can’t continue shooting. canShoot is set to false when the timer runs out, with the timer itself using the update method to tick down the time and when it hits zero it calls a method in the shoot script called PlayerShooting which uses a taggable bool and sets it to false making it so the player can no longer shoot.

When it comes to this project I think that the biggest issue I encountered with it was the limitations of the design I had for the game which made it harder for me to find useful guides and caused me to rely heavily on other people and chat GPT to figure out how to get the code working and create a functional game. Overall, I am happy with how my game turned out it functions and its actually fun to play. I do think one day if/when I have more free time and more skill I might come back to this project and tinker a little with it, maybe add audio or another game mode, perhaps upgrade the graphics and just make it a more complete game.
